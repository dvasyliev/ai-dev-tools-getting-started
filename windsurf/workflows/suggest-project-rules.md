# Suggest Project Rules

Analyze the frontend project and suggest a clear set of project rules for coding standards, naming, structure, and consistency.

1. Analyze the codebase structure, component patterns, file naming, folder naming, and import style.
2. Review examples of components, hooks, tests, styles, and Storybook stories if they exist.
3. Detect repeated patterns and implicit conventions already used in the project.
4. Suggest practical project rules for:
   - coding style
   - component structure
   - file and folder naming
   - hooks naming
   - test file naming
   - styles organization
   - exports and imports
5. Keep the rules specific, short, and based on the actual project.
6. Format the result as a markdown rules template that can be reviewed and reused.
