# Review Frontend Accessibility

Analyze the frontend UI and suggest accessibility improvements based on common real-world issues.

1. Analyze the relevant components or pages.
2. Check for common accessibility problems such as:
   - missing labels
   - poor button semantics
   - missing alt text
   - keyboard navigation issues
   - weak focus handling
   - unclear headings
   - low contrast risks if visible in code or styles
3. Identify which issues affect usability most.
4. Suggest specific improvements with clear reasoning.
5. If possible, propose small code changes to improve accessibility.
6. Summarize the most important accessibility fixes first.
