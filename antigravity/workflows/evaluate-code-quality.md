# Evaluate Code Quality

## Evaluation Criteria

Evaluate the code and give a score from 0-10 for each section:

1. Functionality: Does the code correctly solve the task and handle main states?
2. Code Clarity: Is the code easy to read and understand?
3. Structure: Is the code organized well and aligned with project patterns?
4. Code Duplication: Is repeated logic or UI avoided?
5. Performance: Are obvious performance issues avoided?
6. Accessibility: Are there any visible accessibility problems?
7. Tests Coverage: Are important scenarios covered by tests?
8. Maintainability: Will this code be easy to change later?

## Output Format

- For each section, write a short comment what is good and what should improve, and give a score in this format: X/10.
- At the end, provide the overall score in this format: X/10 with a brief summary.
- Suggest the top 3 critical fixes first.
