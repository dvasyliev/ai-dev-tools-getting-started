# Write Unit Tests for a Component

Analyze a frontend component and create unit tests that cover expected behavior, negative scenarios, and edge cases.

1. Ask which component should be tested if it is not clear.
2. Analyze the component implementation, props, states, conditional rendering, and interactions.
3. Identify the expected positive scenarios.
4. Identify negative scenarios, such as missing data, invalid props, disabled states, or error states.
5. Identify edge cases, such as empty arrays, long text, null values, undefined values, loading states, or boundary conditions.
6. Write unit tests following the existing project testing style and tools.
7. Make sure the tests are readable and grouped logically.
8. Briefly explain what scenarios are covered and what is still not covered.
